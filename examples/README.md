# Example Files

This directory contains sample SBOM files, configuration files, and test data for demonstration and testing purposes.

## Sample SBOM Files

### `sample_sbom.json`
Comprehensive multi-runtime test SBOM:
- **Format**: CycloneDX specification
- **Components**: 33 components across all supported runtimes
- **Runtimes**: Python, Java, Node.js, .NET, Ruby
- **Includes**: Linux packages, ISV software, open source libraries
- **Use case**: Complete end-to-end testing of all runtime analyzers

### `sample_cyclonedx.json`
Basic CycloneDX format SBOM:
- **Format**: CycloneDX specification
- **Use case**: Testing CycloneDX parser functionality

### `sample_spdx_sbom.json`
Extended SPDX format with additional metadata:
- **Format**: SPDX with complete component information
- **Use case**: Testing comprehensive SPDX parsing

### `sample_spdx.json`
Basic SPDX format SBOM:
- **Format**: SPDX specification
- **Use case**: Testing SPDX parser functionality

### `sample_app_identifier_cyclonedx_sbom.json`
Real-world CycloneDX SBOM generated by app_identifier tool:
- **Format**: CycloneDX with system and application packages
- **Components**: Mix of system packages and application dependencies
- **Use case**: Testing realistic application analysis scenarios

## Configuration Files

### `runtime_config.yaml`
Runtime analysis configuration:
- **Purpose**: Configure runtime testing behavior
- **Settings**: Container options, test parameters, timeout values
- **Use case**: Customize runtime analysis execution

### `sample_knowledge_base.json`
Example knowledge base with compatibility data:
- **Software entries**: Sample package compatibility information
- **Version ranges**: Version compatibility examples
- **Use case**: Testing knowledge base functionality

## Test Data

### `test-pom.xml`
Sample Maven POM file:
- **Purpose**: Testing Java dependency analysis
- **Use case**: Direct Maven manifest analysis

### `JARs/aws-java-sdk-ec2-1.12.792.jar`
Sample JAR file:
- **Purpose**: Testing JAR analysis functionality
- **Use case**: Direct JAR file dependency extraction

## Usage Examples

### Basic SBOM Analysis
```bash
# Analyze comprehensive test SBOM (all runtimes)
python graviton_validator.py examples/sample_sbom.json

# Analyze CycloneDX SBOM
python graviton_validator.py examples/sample_cyclonedx.json

# Analyze SPDX SBOM
python graviton_validator.py examples/sample_spdx_sbom.json

# Analyze with custom knowledge base
python graviton_validator.py -k examples/sample_knowledge_base.json examples/sample_app_identifier_cyclonedx_sbom.json
```

### Runtime Analysis
```bash
# With runtime testing (comprehensive test)
python graviton_validator.py --runtime --test --containers examples/sample_sbom.json

# With custom runtime config
python graviton_validator.py --runtime --test --runtime-config examples/runtime_config.yaml examples/sample_spdx_sbom.json
```

### Direct Manifest Analysis
```bash
# Analyze Maven POM
python graviton_validator.py --runtime-only java --input-file examples/test-pom.xml

# Analyze JAR file
python graviton_validator.py --jars examples/JARs/
```

### Output Formats
```bash
# JSON output
python graviton_validator.py -f json -o report.json examples/sample_sbom.json

# Excel output
python graviton_validator.py -f excel -o report.xlsx examples/sample_sbom.json

# Markdown output
python graviton_validator.py -f markdown -o report.md examples/sample_spdx_sbom.json
```

## Testing Scenarios

### Parser Validation
```bash
# Test all SBOM formats
python graviton_validator.py examples/sample_cyclonedx.json
python graviton_validator.py examples/sample_spdx.json
python graviton_validator.py examples/sample_spdx_sbom.json
```

### Multi-Runtime Testing
```bash
# Full analysis with all runtimes
python graviton_validator.py --runtime --test --containers examples/sample_sbom.json

# Batch processing
python graviton_validator.py examples/sample_*.json
```
